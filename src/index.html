Why this Deviation?

Initially, I was looking for a script to deploy all assets of TailwindCSS methodically. I stumbled on Manjunath's excellent gulp script. The first question that came to my mind was, why doesn't the original TailwindCSS distribution include a script like this? Well, the changes made for the production environment have deviated so far from the intent of Manjunath's original work that this fork will likely be a permanent feature.
...

Typescript is now part of the compilation process. During the build stages for production, all HTML and custom scripting is minified.

The build process (yarn build) cleans and creates a brand new destination directory instead of utilizing assets built during the development stage (yarn dev).

Both script and style assets from node modules can easily be added as part of the build process and minified into a single file.

As demonstrated in the original script, all image assets are optimized during the deployment state (yarn build).

Usage

Use the following commands to get started.

// To Install Dev Depedencies
npm install (or) yarn install

// To start development and server for live preview
npm run dev (or) yarn dev

// To generate minifed,optamized files for production server
npm run prod (or) yarn build

Configuration

To change the path of files and destination/build folder configure at config.js file

config: { ... port: 9050 //Update port here }, paths: { root: "./", src: { base: "./src", css: "./src/css", js: "./src/js", img: "./src/img" }, dev: { base: "./public_html", css: "./public_html/css", js: "./public_html/js", img: "./public_html/img" }, dist: { base: "./public_html", css: "./public_html/css", js: "./public_html/js", img: "./public_html/img" } }

Other Settings

These settings can be changed in the following file gulpfile.js file.
Make sure `yarn dev` is executed again after changes.
